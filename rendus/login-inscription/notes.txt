Remarques générales 

- Bien dans l'ensemble
- Beau code, pas trop de problème d'indentation
- Beaucoup de volontaires
- Export SQL dans fichier .sql
- Essayer de garder la config avec defines (pour correction)
- Fichier README avec explications, listing des features, problèmes rencontrés (éviter de mettre dans mail)
- Bien commenter (même les parties du code foireuses)
- Bien pour les versions en ligne 
- Beaucoup de notices, trouver comment les faire apparaître 
- Éviter de mettre plusieurs CSS, mettez une classe sur le body
- "exit" après "header('Location:...')"
- Pour certains ça manque d'intégration. Ce n'est pas parce que c'est PHP qui compte le plus qu'il ne faut pas montrer qu'on sait encore faire du front.
- Ne pas faire de echo en plein dans le PHP
- Ne pas fermer PHP s'il n'y a que du PHP dans le fichier
- Enlever les debugs
- Bien parti pour la majorité, il faut persister
- Ne faites pas de "echo" en plein milieu du PHP avant le HTML. Il faut sauvegarder la donnée dans une variable pour la restituer plus tard dans le HTML au bon endroit.


Features

- Gestion des erreurs du formulaire
- Demander deux fois le mot de passe à l'inscription et tester que les deux soient égaux
- Tester la complexité du mot de passe
- Tester si le mail n'a pas déjà été enregistré dans la BDD
- Plus de champs à l'inscription
- Envoyer un mail de confirmation d'inscription
- Envoyer un mail contenant un lien permettant de confirmer l'inscription
- Bouton de déconnexion sur la page privée
- Système "Mot de passe oublié"
- Système "Se souvenir de moi"
- Captcha
- Limiter le nombre de tentatives
- Mots de passe sécurisés (hash et salt minimum)
